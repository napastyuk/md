## Для чего нужен doctype
Что бы определить версию html , которая используется в Документе. То введения doctype не было единого принятого стандарта html и разные браузеры парсили html с разными правилами. Для введения строго стандарта html был придуман xtml. Для того что бы отличать (на тот момент) старые нестрогие html от строгих был введён doctype в котором прописывалась xml схема, которой соответствовал html

## Почему xml потерял популярность?
1. Строгие правила относительно иерархии тегов приводили к тому что любая неаккуратная работа с DOM (в частности средствами js), делала страницу не валидной и страница переставала показывается целиком, а не только ошибочный блок. 
2. Для потоковой передачи не просто документов а однотипных данных (AJAX, RSS) Доля разметки в преобладала над полезными данными, что создавала дополнительную нагрузку на сеть и преобразования
3. Вложенная структура разметки была подвержена багам (например рекурсивно вложенные данные)

## Какая последняя версия html?
Вопрос не совсем корректен. Версионирование по номерам прекратилось примерно c hml5. Потому что 
1. Не было единого стандарта который бы описывал все детали html5. Была группа спецификаций объединённая под общим маркетинговым брендом html5.
2. Все браузеры только выборочно поддерживали даже существующие стандарты html тегов.
Поэтому было решено итеративно улучшать отдельные спецификации, в соответствии с договорённостями внутри w3c не объединяя их вместе под одной версией.
