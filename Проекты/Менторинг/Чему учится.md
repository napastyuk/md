### Этап 2. BPMN и моделирование процессов
**Цель этапа**: научится записывать процессы любой сложности на бумагу. Так чтобы это было всем понятно.
**Книги:** Брюс Сильвер — BPMN Method and Style
На русском полностью найти сложно, но даже переведенные конспекты очень полезны.
**Видео на Youtube, искать по названиям и темам:**
- BPMN для начинающих
- BPMN 2.0 за 2 часа
- Моделирование бизнес-процессов BPMN

Здесь важнее практика, чем теория, поэтому все пробуем в Draw.io / diagrams.net или любом другом редакторе BPMN. Что бы было что моделировать, можно записывать процессы из медицины:
- запись пациента;
- госпитализация;
- выписка;
- лабораторные исследования

### Этап 3 ПЕТ-проект
**Цель этапа**: закрепить на практике новую информацию, и сделать что-то что можно показывать на собеседованиях в качестве доказательств компетенций.  
**Надо:** Заводим github или gitverse чтобы было куда выкладывать. Цель не выложить код, а показать навыки документирования моделирование процессов. Выбираем любой сложный процесс, можно твою систему с вебинарами, и записываем:
1. BPMN diagrams  
2. User stories  
3. Backlog
4. Requirements
5. Database model (после этапа SQL)  
6. API description (после этапа API и интеграции) 

Процесс поиска работы будет не быстрый, поэтому сразу после этапа 3 (ПЕТ-проект) начинаем откликаться на вакансии. Начать можно с базовых типа такой https://spb.hh.ru/vacancy/133567951 
Как работать с вакансиями тема отдельной статьи.
